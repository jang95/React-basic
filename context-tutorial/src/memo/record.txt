Context API는 전역적으로 사용할 데이터가 있을 때 유용한 기능
ex) 로그인 정보, 애플리케이션 환경 설정, 테마 등

리덕스, 리액트 라우터, styled-components 등의 라이브러리는 Context API를 기반으로 구현

쓰는 이유
react의 경우 전역적으로 필요한 상태를 관리할 때는 최상위 컴포넌트에서 props로 데이터를 전달하는데
최하위 컴포넌트로 갈수록 거쳐가는 컴포넌트의 수가 많아지면 유지 보수성이 떨어지기 때문에
Context API를 통해 데이터가 컴포넌트를 거쳐가는 일 없이 한 번에 원하는 값을 받아와 사용 가능
